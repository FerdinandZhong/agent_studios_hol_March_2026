Six-Stage Agentic AI Pipeline — Trade Fraud Detection

From scanned invoice to evidenced investigation brief — in seconds

📄
Input
Scanned invoice image or PDF
⚡

Confidence Propagation — runs through every stage

Agent 1 assigns a per-field OCR confidence score (0–1). Fields below 0.80 are added to a low_confidence_fields[] list that travels with the data. Every downstream agent appends [LOW CONFIDENCE — verify manually] to any finding that relies on a flagged field. Uncertainty is never silently absorbed — it surfaces explicitly in the final investigation brief.

Agent 1
Document Analyst
OCR Extraction & Field Mapping
Actions
  • Run AI vision OCR on uploaded document
  • Extract per-line text with confidence scores
  • Map 12 canonical trade fields (HS code, prices, entities, route)
  • Flag fields below 0.80 confidence threshold
Tool
AI Vision OCR
Output
Structured fields JSON
+ low_confidence_fields[ ] list propagated to all downstream agents
Agent 2
Price Validator
Price Anomaly Detection
Actions
  • Query price benchmark for HS code + origin country
  • Pull 100 recent comparable transactions
  • Retrieve entity transaction baseline (shipper/consignee)
  • Compute z-score and normalised anomaly score (0–1)
  • Flag UNDER_INVOICING if z-score < −2.0 or price < 30% benchmark
Tool
SQL — Trade History DB
Output
Price anomaly score · UNDER_INVOICING flag
Weight: 25% of composite score
Agent 3
Compliance Auditor
Regulatory Breach Detection
Actions
  • Query import requirements, quotas, and restrictions for HS code + route
  • Verify Rules of Origin and preferential duty eligibility
  • Assess transshipment and origin-washing risk
  • Cross-check required certificates against extracted document fields
  • Flag missing certificates (e.g. Form A for GSP rates)
Tool
RAG — Compliance KB
Output
Breach list · compliance risk score
Weight: 20% of composite score
Agent 4
Intelligence Analyst
Entity & Sanctions Screening
Actions
  • Semantic search: shipper, consignee, broker vs. fraud bulletins
  • Sanctions and watchlist screening
  • LLM alias/transliteration assessment for near-matches
  • Screen trade route vs. known fraud corridors for declared HS code
Tool
RAG — Intelligence KB
Output
Entity match results · intelligence score
Weight: 30% of composite score
Agent 5
Network Analyst
Collusion & Scheme Detection
Actions
  • Inspector–broker co-occurrence: flag pairs with >10 shared clearances
  • Trading network clustering: map price patterns across connected entities
  • Split-shipment detection: multiple low-value declarations within 90 days from same pair
Tool
SQL — Trade History DB
Output
Collusion indicators · collusion risk score
Weight: 25% of composite score
Agent 6
Risk Officer
Score Aggregation & Report
Actions
  • Aggregate scores from all four specialist agents
  • Compute weighted composite risk score
  • Classify risk tier (LOW / MEDIUM / HIGH / CRITICAL)
  • Assemble evidence chain and per-module findings
  • Add mandatory human-review block for low-confidence findings in HIGH/CRITICAL cases
  • Generate signed PDF investigation brief
Tool
PDF Generator
Output
Risk tier · PDF investigation brief
Case summary · score table · evidence chain · extraction quality · approval block
Composite Risk Score Formula
Score = (Price Anomaly × 25%) + (Compliance × 20%)
+ (Intelligence × 30%) + (Collusion × 25%)
LOW  0.0–0.30 MEDIUM  0.30–0.60 HIGH  0.60–0.80 CRITICAL  0.80–1.00
